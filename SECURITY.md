# Security Policy

## Supported Scope
Security reports are accepted for the current default branch of this repository.

## Reporting a Vulnerability
If you discover a security issue, do not open a public issue first.

Please report privately with:
- Affected area and impact
- Reproduction steps
- Proof-of-concept details (if available)
- Suggested mitigation (optional)

Use the project contact channel listed by the repository owner.

## Response Targets
- Initial acknowledgement: within 5 business days
- Triage decision: as soon as reasonably possible based on severity
- Fix timeline: severity- and risk-dependent

## Disclosure
- Please allow time for verification and remediation before public disclosure.
- Coordinated disclosure is preferred.

## Out of Scope
- Reports based only on outdated dependencies without exploit path
- Non-actionable hardening suggestions without concrete risk
- Social engineering or physical attacks
